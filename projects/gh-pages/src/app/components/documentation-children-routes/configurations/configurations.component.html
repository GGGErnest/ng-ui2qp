Ng-Ui2Qp configurations

We are aware that every use case could be different, and that's why we've provided some configurations that should help to adjust the library features to a wide variety of use cases.

Global configurations

The Global Configurations are the ones provided when importing UI2QpModule their scope is within the module that imported Ui2QpModule.

Configuration Object

This is how the configuration object looks like:

autoUpdating

Property that groups the configurations specific to the auto-updating feature.

replaceState

Defines whether updating the URL due to a change in the model adds or replaces the state in the browser's history. Check this for more info about it

logLevel

Logging level to be output for the logging service. The possible values are:

cryptoSecretKey

Secret key used in the "encrypted" serializer and deserializers. If not provided a default value will be used.

Default values

Global configurations default values if not provided.

UI2QpRoot Configurations

Ui2QpRoot accepts some settings which define how it should behave. Those are the Global Configurations explained above. The configurations won't be needed to pass in when creating a Ui2QpRoot with the Ui2QpBuilder service. It injects by default the global configurations when creating a Ui2QpRoot instance.

UI2QpGroup Configurations

The UI2QpGroup object can be configured to adjust its behavior to specific use cases, those configurations are provided to the Ui2QpBuilder service when creating an instance of it. Following are those configurations:

qpName

Set the Qp's name for this Ui2QpGroup instance.

validatorOrOpts and asyncValidators

Both properties have the same function that in ReactiveForms. Check Angular documentation to know more about them.

Ui2QpControl Configurations

Ui2QpControl configurations when created with the Ui2QpBuilder:

qpName

Set the Qp's name for this Ui2QpControl.

validatorOrOpts and asyncValidators

They have the same function that the similar properties in the Angular's FormGroup class. Check Angular documentation to know more about them.

type

This property is used to assign the right serializer and deserializer to the control and if not provided then a default value of "string" will be used. Please checkout the serializer and a deserializerdocumentation to know more about how they affect the Ui2QpControl's value when set it or get it.

defaultVal

The value to be used when is null the value to set. We know that a Ui2QpControl instance binds a model property value to a Qp. By default, if a Qp value is "null", it will be removed from the URL, that avoids useless URL segments such as ?q= (empty value). However, in some cases you might want this "default" to be a different value, e.g., a numeric input should perhaps default to 0 instead. Look at the next example that shows how to set the default value.
