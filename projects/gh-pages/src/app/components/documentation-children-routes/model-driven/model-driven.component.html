Overall view

In general, using this approach generally is only a matter of some few steps:

  1. Import Ui2QpModule.
  2. Create a model that defines the Qps.
  3. Create a Ui2QpRoot object.
  4. Bind the model to the template.

Import Ui2QpModule and configure it

Importing Ui2QpModule

This can be done in two places:

  • AppModule: If imported in the AppModule all the library features will be available across the whole App.

  • Module import: The Ui2QpModule could also be imported in feature modules making ng-ui2qp's features available only within that module.

IMPORTANT: We want to point out that when importing Ui2QpModule some services are provided by default, making them automatically available to use, but none of them are provided in the root injector by default, they will be available at the injection tree level the model was provided.

Create a model

In this step, we'll create a model(Ui2QpGroup) that defines the Qps and will be bonded to the UI components in the template. Each property in the model contributes to how the Qps are going to be named by default, but they can be renamed if desired.

Control's type

When creating a model is normal to have different types of data, and in the cases where it's not "string" the type hast to be specified. Take a look in the previous example at the "houseNumber" property within the "address" group, it's a number, then we set the property "type" to "number" within the configurations passed into the Ui2QpBuilder, it will assign a serializer and a deserializer of type "number" to the Ui2QpControl, guaranteeing that the model and the Qps receive the right type of value.

Providing default values

By default, if a Qp's value is null, it will be removed from the URL, which avoids useless URL segments such as ?q= (empty value). However, in some cases, you might want this "default" to be a different value, e.g., a numeric input should perhaps default to 0 instead.

The way of achieving this is to include in the configuration object mentioned before another property called "defaultValue" with the desired default value.
