Overall view

This approach in general requires very few steps, here are they:

  1. Import Ui2QpModule.
  2. Use Ui2qpControl directive on the template.

Import Ui2QpModule and configure it

Importing Ui2QpModule

This can be done in two places:

  • AppModule: If imported in the AppModule all the library features will be available across the whole App.

  • Module import: The Ui2QpModule could also be imported in feature modules making ng-ui2qp's features available only within that module.

IMPORTANT: We want to point out that when importing Ui2QpModule some services are provided by default, making them automatically available to use, but none of them are provided in the root injector by default, they will be available at the injection tree level the model was provided.

Using the Ui2qpControl directive

Requirements

Before using the Ui2qpControlDirective, is important to know that the host element must implement a suitable ControlValueAccessor because it's the way the directive uses to interact with the host element state(value). If the host element doesn't have a ControlValueAccessor implementation, the Ui2qpValueAccessorDirective can be used to provide an ad-hoc control value accessor by wiring it up in the template directly. The ControlValueAccessor is a helper directive created to help in these specific cases. We have taken this idea from ngqp. See in the next sections how to use it.

Configurations

The directive provide a set of configurations that can be passed in as inputs, check this link to know more about it.
